# チャットボット 設計書

## 1. 概要

| 項目           | 内容                                                |
| -------------- | --------------------------------------------------- |
| 目的           | 個人学習・実験用のシンプルなチャットボット          |
| コスト方針     | GitHub Copilot 加入済みプランを活用（追加費用なし） |
| 利用環境       | ブラウザ                                            |
| AIエンジン     | GitHub Models API（gpt-4o-mini）※ OpenAI 互換       |
| バックエンド   | Node.js + Express                                   |
| フロントエンド | HTML / CSS / JavaScript（1ファイル）                |
| コンテナ       | Docker / Docker Compose                             |

---

## 2. ディレクトリ構成

```
chatbot/
├── Dockerfile             # Node.js アプリのコンテナ定義
├── docker-compose.yml     # コンテナ起動設定
├── server.js              # Express サーバー本体
├── package.json           # 依存パッケージの定義
├── .env                   # APIキーなどの秘密情報（Gitに含めない）
├── .gitignore
└── public/
    └── index.html         # チャット画面（フロントエンド）
```

---

## 3. 機能一覧

| 機能             | 説明                                                   |
| ---------------- | ------------------------------------------------------ |
| チャット送信     | ユーザーがメッセージを入力して送信する                 |
| AI応答表示       | OpenAI APIの返答を画面に表示する                       |
| 会話履歴保持     | 直近の会話をサーバー側のメモリで保持し、文脈を維持する |
| ローディング表示 | API応答待ち中にローディングを表示する                  |
| レートリミット   | 短時間の連打ちを防ぐ（コスト削減）                     |
| 文字数制限       | 入力・出力のトークン数を制限（コスト削減）             |

---

## 4. システム構成図

```
[ブラウザ]
   ユーザー入力
      │
      │ POST /api/chat  (JSON)
      ▼
[Node.js + Express サーバー]
   会話履歴を管理
      │
      │ GitHub Models API 呼び出し
      │ (OpenAI SDK / Azure エンドポイント)
      ▼
[GitHub Models]
   gpt-4o-mini で返答を生成
      │
      │ レスポンス (JSON)
      ▼
[Node.js + Express サーバー]
      │
      │ レスポンス返却
      ▼
[ブラウザ]
   返答を画面に表示
```

## 5. API 仕様

### POST /api/chat

#### リクエスト

```json
{
  "message": "ユーザーが入力したテキスト"
}
```

#### レスポンス（正常時）

```json
{
  "reply": "AIの返答テキスト"
}
```

#### レスポンス（エラー時）

```json
{
  "error": "エラーメッセージ"
}
```

## 6. データフロー（会話履歴）

- 会話履歴はサーバーのメモリ上（配列）で管理する
- 使用モデル：`gpt-4o-mini`（GitHub Models 経由）
- システムプロンプト：汎用（「You are a helpful assistant.」）
- 最大履歴件数：**直近6件**（日常利用しやすいバランス）
- `max_tokens`：**350**（回答品質とコストのバランス）
- 1日上限：**100リクエスト**（日常利用向けライト設定）
- レートリミット：**10秒に1リクエスト**
- サーバー再起動で履歴はリセットされる（お試し版のため永続化なし）

---

## 7. コスト最適化の方針

### GitHub Models の利用

GitHub Copilot 加入済みの場合、**GitHub Models** 経由で OpenAI 互換の API を追加費用なしで利用できる。

| 項目           | 内容                                        |
| -------------- | ------------------------------------------- |
| エンドポイント | `https://models.inference.ai.azure.com`     |
| 認証           | GitHub Personal Access Token                |
| SDK            | openai パッケージをそのまま使用可能（互換） |

### トークン節約対策

| 対策                     | 内容                            | 実装場所     |
| ------------------------ | ------------------------------- | ------------ |
| モデルの選択             | `gpt-4o-mini`（軽量・低コスト） | `server.js`  |
| `max_tokens` 制限        | 出力トークンを350までに制限     | `server.js`  |
| 履歴件数制限             | 直近6件のみ送信                 | `server.js`  |
| 1日利用上限制限          | 1日100リクエストまで            | `server.js`  |
| 入力文字数制限           | 200文字以上は送信しない         | `index.html` |
| サーバー側レートリミット | 10秒に1リクエストまで           | `server.js`  |
| 連打ち防止               | 応答中は送信ボタンを無効化      | `index.html` |

## 9. 環境変数（.env）

```
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxx
PORT=3000
MAX_HISTORY_MESSAGES=6
MAX_OUTPUT_TOKENS=350
DAILY_REQUEST_LIMIT=100
RATE_LIMIT_WINDOW_SECONDS=10
RATE_LIMIT_MAX_REQUESTS=1
```

> GitHub Personal Access Token は https://github.com/settings/tokens で発行する。  
> スコープは **読み取りのみ**（`read:user` 程度）で十分。  
> `.env` は `docker-compose.yml` の `env_file` 設定でコンテナに渡す。

## 9b. AIエンジン 比較（教える場合の参考）

|                      | GitHub Models                      | Google Gemini API                  |
| -------------------- | ---------------------------------- | ---------------------------------- |
| **無料枠**           | Copilot 加入者向けに無料           | 無料枠あり（Gemini 2.0 Flash）     |
| **API キー取得**     | GitHub Token（既存アカウントでOK） | Google アカウントで取得            |
| **OpenAI 互換**      | ✅ openai SDK がそのまま使える     | ❌ SDK が異なる（`@google/genai`） |
| **モデル**           | gpt-4o-mini など                   | gemini-2.0-flash など              |
| **非エンジニア向け** | 〇 GitHub 1つで完結                | ◎ Google アカウントで手軽          |
| **コード変更量**     | 少ない（エンドポイントだけ変更）   | 中程度（SDK と呼び出し方が変わる） |

> **教える場合のおすすめ**: Google アカウントさえあれば使える **Google Gemini API** が導入ハードルが最も低い。

## 10. Docker 起動手順

```bash
# 初回またはイメージ再ビルド時
docker compose up --build

# 2回目以降
docker compose up

# 停止
docker compose down
```

ブラウザで `http://localhost:3000` にアクセスして使用する。

## 11. セキュリティ考慮事項

- `.env` を `.gitignore` に追記し、APIキーをGitに含めない
- CORS はローカル開発のみ許可（本番公開時は要見直し）
- ユーザー入力のサニタイズはフロントエンド側で基本的なバリデーションを実施
